# Plan 6.4: Eval Runner and Database Persistence

<objective>
To implement the master runner script that initiates the evaluations, writes the output matrices to the `.EvalRun` Prisma model for dashboard visibility, and enforces hard accuracy thresholds to act as a CI gate.

Purpose: To convert the isolated Eval suites into a continuous feedback mechanism that persists metrics and safeguards regression.
Output: A runnable `runAllEvals` file executable via a simple `npm run test:eval`.
</objective>

<context>
Load for context:
- docs/mastra-patterns.md
- prisma/schema.prisma
- src/evals/scenarios/compliance.eval.ts
- src/evals/scenarios/onboarding.eval.ts
</context>

<tasks>

<task type="auto">
  <name>Implement Master Eval Runner</name>
  <files>src/evals/run.ts</files>
  <action>
    Import `runEvals` from `@mastra/evals`. Feed it `complianceEvals` and `onboardingEvals`.
    After computation, insert a `prisma.evalRun.create()` record containing `overallScore`, `scenarioResults`, and `scorerBreakdown`.
    Enforce `if (results.overallScore < 0.80) throw new Error(...)` to fail CI.
    AVOID: Proceeding or silently logging failures; if thresholds are missed, the process MUST crash.
  </action>
  <verify>npx tsc src/evals/run.ts --noEmit</verify>
  <done>Runner aggregates data, saves to DB, and enforces minimum accuracy.</done>
</task>
