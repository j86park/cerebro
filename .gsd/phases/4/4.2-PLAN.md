# Plan 4.2: Compliance Tool Factories

## Objective
Create the 5 compliance-specific tool factories that power the Compliance Agent's escalation ladder and document monitoring capabilities.

## Context
- docs/mastra-patterns.md (sections 4, 5 — Escalation Tool Pattern)
- src/lib/db/vault-service.ts
- prisma/schema.prisma (DocumentStatus, DocumentType, DocumentCategory enums)
- src/lib/documents/registry.ts (document metadata)
- src/lib/config.ts (env.DEMO_DATE, env.DRY_RUN)

## Tasks

<task type="auto">
  <name>Create getDocumentComplianceStatus Tool</name>
  <files>src/tools/compliance/getDocumentComplianceStatus.ts</files>
  <action>
    Create `buildGetDocumentComplianceStatus(vault: VaultService)` tool factory.
    - **Type:** Observation (read-only)
    - **inputSchema:** `{ includeValid: z.boolean().default(false) }`
    - **outputSchema:** documents array with urgency levels + summary with counts and highestUrgency
    - **execute:** Calls `vault.getDocuments()`, calculates `daysUntilExpiry` from `env.DEMO_DATE`, maps urgency levels (CRITICAL/HIGH/MEDIUM/LOW/NONE), filters by `includeValid`.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool computes urgency correctly from DEMO_DATE.</done>
</task>

<task type="auto">
  <name>Create sendClientReminder Tool</name>
  <files>src/tools/compliance/sendClientReminder.ts</files>
  <action>
    Create `buildSendClientReminder(vault: VaultService)` tool factory.
    - **Type:** Action (write + external effect)
    - **inputSchema:** `{ documentId: z.string(), subject: z.string(), body: z.string(), reasoning: z.string().min(20) }`
    - **outputSchema:** `{ success: boolean, dryRun: boolean, notificationCount: number }`
    - **execute:** Checks DRY_RUN. Sends email if not dry run. Increments document `notificationCount` via `vault.updateDocumentStatus()`. Always logs via `vault.logAction()`.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool compiles, increments notification count, and logs.</done>
</task>

<task type="auto">
  <name>Create escalateToComplianceOfficer Tool</name>
  <files>src/tools/compliance/escalateToComplianceOfficer.ts</files>
  <action>
    Create `buildEscalateToComplianceOfficer(vault: VaultService)` tool factory.
    - **Type:** Escalation (prerequisite enforcement)
    - **inputSchema:** `{ reasoning: z.string().min(20) }`
    - **outputSchema:** `{ success: boolean, dryRun: boolean }`
    - **execute:** Self-enforces prerequisite: checks `vault.getActionHistory()` for at least 2 `SEND_CLIENT_REMINDER` actions before allowing escalation. Throws descriptive error if prerequisites not met. Always logs.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool enforces escalation prerequisites.</done>
</task>
