# Plan 4.3: Onboarding Tool Factories

## Objective
Create the 6 onboarding-specific tool factories that power the Onboarding Agent's staged document collection pipeline.

## Context
- docs/mastra-patterns.md (sections 4, 5)
- src/lib/db/vault-service.ts
- src/lib/documents/onboarding-stages.ts (stage -> document type mapping)
- prisma/schema.prisma (OnboardingStatus, DocumentType enums)
- src/lib/config.ts (env.DEMO_DATE, env.DRY_RUN)

## Tasks

<task type="auto">
  <name>Create getOnboardingStatus Tool</name>
  <files>src/tools/onboarding/getOnboardingStatus.ts</files>
  <action>
    Create `buildGetOnboardingStatus(vault: VaultService)` tool factory.
    - **Type:** Observation (read-only)
    - **inputSchema:** empty `z.object({})`
    - **outputSchema:** current stage, stage name, required documents for current stage (with status), completion percentage, overall onboarding status
    - **execute:** Calls `vault.getClientProfile()` for stage info and `vault.getDocuments()` for document statuses. Cross-references with onboarding stage requirements.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool correctly maps stage to required documents.</done>
</task>

<task type="auto">
  <name>Create requestDocument Tool</name>
  <files>src/tools/onboarding/requestDocument.ts</files>
  <action>
    Create `buildRequestDocument(vault: VaultService)` tool factory.
    - **Type:** Action (write + external effect)
    - **inputSchema:** `{ documentType: z.string(), message: z.string(), reasoning: z.string().min(20) }`
    - **outputSchema:** `{ success: boolean, dryRun: boolean }`
    - **execute:** Checks DRY_RUN. Sends document request email if not dry run. Updates document status to `REQUESTED` via `vault.upsertDocument()`. Always logs.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool creates/updates document record and logs.</done>
</task>

<task type="auto">
  <name>Create validateDocumentReceived Tool</name>
  <files>src/tools/onboarding/validateDocumentReceived.ts</files>
  <action>
    Create `buildValidateDocumentReceived(vault: VaultService)` tool factory.
    - **Type:** Observation + Action (validates and updates)
    - **inputSchema:** `{ documentId: z.string() }`
    - **outputSchema:** `{ valid: boolean, documentType: string, status: string, notes: string }`
    - **execute:** Looks up the document, checks it exists and belongs to this client's vault. Returns validation result. Does NOT auto-update status (agent decides).
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool validates without auto-mutating.</done>
</task>

<task type="auto">
  <name>Create advanceOnboardingStage Tool</name>
  <files>src/tools/onboarding/advanceOnboardingStage.ts</files>
  <action>
    Create `buildAdvanceOnboardingStage(vault: VaultService)` tool factory.
    - **Type:** Action (write, prerequisite enforcement)
    - **inputSchema:** `{ reasoning: z.string().min(20) }`
    - **outputSchema:** `{ success: boolean, previousStage: number, newStage: number }`
    - **execute:** Self-enforces: checks all required documents for the current stage have `VALID` status. Throws if any are missing/invalid. Calls `vault.resetOnboarding()` to increment stage. Always logs.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool enforces stage advancement prerequisites.</done>
</task>

<task type="auto">

  <action>
