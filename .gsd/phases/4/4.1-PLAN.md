# Plan 4.1: Shared Tool Factories

## Objective
Create the 4 shared tool factories used by both agents. These tools provide read access to client profiles and action history, as well as write access for logging actions and sending advisor alerts.

## Context
- docs/mastra-patterns.md (sections 4, 5, 12, 13)
- src/lib/db/vault-service.ts
- prisma/schema.prisma (ActionType, AgentType, TriggerType enums)

## Tasks

<task type="auto">
  <name>Create getClientProfile Tool</name>
  <files>src/tools/shared/getClientProfile.ts</files>
  <action>
    Create `buildGetClientProfile(vault: VaultService)` tool factory.
    - **Type:** Observation (read-only)
    - **inputSchema:** empty `z.object({})`
    - **outputSchema:** client profile shape (id, name, email, accountType, onboardingStatus, onboardingStage, advisor name/email, firm name)
    - **execute:** Calls `vault.getClientProfile()` and returns the client profile.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool compiles and follows the canonical factory pattern.</done>
</task>

<task type="auto">
  <name>Create getActionHistory Tool</name>
  <files>src/tools/shared/getActionHistory.ts</files>
  <action>
    Create `buildGetActionHistory(vault: VaultService)` tool factory.
    - **Type:** Observation (read-only)
    - **inputSchema:** `z.object({ limit: z.number().default(20) })`
    - **outputSchema:** array of action log entries (id, agentType, actionType, trigger, reasoning, outcome, performedAt, nextScheduledAt)
    - **execute:** Calls `vault.getActionHistory()` and returns sliced to limit.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool compiles and follows the canonical factory pattern.</done>
</task>

<task type="auto">
  <name>Create logAction Tool</name>
  <files>src/tools/shared/logAction.ts</files>
  <action>
    Create `buildLogAction(vault: VaultService)` tool factory.
    - **Type:** Action (write)
    - **inputSchema:** agentType, actionType, trigger, reasoning (required, min 20 chars), outcome (optional), nextScheduledAt (ISO string)
    - **outputSchema:** `{ success: boolean, actionId: string }`
    - **execute:** Calls `vault.logAction(...)` and returns the created action ID.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool compiles and follows the canonical factory pattern.</done>
</task>

<task type="auto">
  <name>Create sendAdvisorAlert Tool</name>
  <files>src/tools/shared/sendAdvisorAlert.ts</files>
  <action>
    Create `buildSendAdvisorAlert(vault: VaultService)` tool factory.
    - **Type:** Action (write + external effect)
    - **inputSchema:** `z.object({ subject: z.string(), body: z.string(), reasoning: z.string().min(20) })`
    - **outputSchema:** `{ success: boolean, dryRun: boolean }`
    - **execute:** Checks `env.DRY_RUN`. If not dry run, sends email via Resend (placeholder for now). Always logs action via `vault.logAction()`.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Tool compiles, respects DRY_RUN, and always logs.</done>
</task>

- [ ] Each tool has strict `inputSchema` and `outputSchema`.
