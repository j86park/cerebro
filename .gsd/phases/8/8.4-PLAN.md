# Phase 8.4 Plan - Simulation Runner & Metrics

## Goal
Execute large-scale simulations and aggregate performance data.

## Tasks
1. **Parallel Orchestration**:
   - Implement batching logic in `orchestrator.ts` to handle 100+ clients across BullMQ concurrency.
2. **Metrics Collection**:
   - Track `timeToResolution`, `complianceScore`, and `remindersSent` per simulation run.
   - Store as JSON in `SimulationRun.metrics`.
3. **Runner Script**:
   - Create `scripts/run-simulation.ts` for headless CLI execution.
4. **Cleanup**:
   - Implement logic to purge simulation data after a run completes (to prevent DB bloat).

## Tests to Implement
1. **Stress Test**: `tests/simulation/orchestration.scale.test.ts`
   - Simulate 100 clients for 1 simulated month.
   - Verify no database deadlocks and all background jobs eventually drain.
2. **Metrics Test**: `tests/simulation/metrics.aggregation.test.ts`
   - Verify `complianceScore` is calculated correctly based on action logs.
   - Verify JSON schema of `SimulationRun.metrics`.

## Verification
- Successful 100-client, 90-day simulation run (Manual integration check).
- Metrics JSON successfully populated in the database.
- `npx vitest tests/simulation/metrics.*` passes.
