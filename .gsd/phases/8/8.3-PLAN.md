# Phase 8.3 Plan - Time Compression Engine

## Goal
Implement the core logic for relative time progression and "ticking" client states.

## Tasks
1. **Config Update**:
   - Add `SIM_TIME_SCALE` to `env` and `config.ts`.
2. **Time-Aware Service**:
   - Refactor `VaultService` or create a wrapper that accepts `simDate`.
3. **Tick Logic**:
   - Implement `SimulationOrchestrator.tick()` which advances the simulation clock.
   - Probability-based event generation (e.g., 5% chance of doc upload per day).
4. **Worker Integration**:
   - Define `SimulationJob` payload in `jobs.ts`.
   - Implement `simulationWorker.ts` in `src/lib/queue/workers/`.

## Tests to Implement
1. **Unit Test**: `tests/simulation/engine.tick.test.ts`
   - Verify `simDate` advances correctly based on configured scale.
   - Mock event probabilities and verify deterministic triggers.
2. **Worker Test**: `tests/simulation/worker.execution.test.ts`
   - Add a simulation job to BullMQ and verify the worker processes it without error.
   - Verify `SimulationRun` record in DB is updated by the worker.

## Verification
- Logs show simulated date advancing 1 day every N real seconds.
- Background jobs are triggered based on simulated events (e.g., document expiry detection).
- `npx vitest tests/simulation/engine.tick.test.ts` passes.
